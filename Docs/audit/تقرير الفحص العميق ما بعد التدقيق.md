إيجاد-البنية-01: الخادم التطويري في بيئة الإنتاج
الدليل: wbgl_server.ps1 السطر 196:


Start-Process -FilePath $phpPath -ArgumentList "-S localhost:$TargetPort server.php"
التفسير: النظام يُشغَّل عبر خادم PHP المدمج (php -S). هذا الخادم:

أُحادي الخيط: يُعالج طلباً واحداً في كل لحظة.
لا يدعم HTTP/2 أو Keep-Alive أو Gzip.
لم يُصمَّم للإنتاج — توثيق PHP يُحذِّر صراحةً من ذلك.
لا Reverse Proxy أمامه (لا Nginx، لا Apache) يفلتر الطلبات الخبيثة.
الخطورة: 🔴 عالية جداً — في حال كان النظام مكشوفاً للشبكة الداخلية أو الخارجية.
التخفيف: نشر النظام خلف Nginx أو Apache مع PHP-FPM.

إيجاد-البنية-02: مجلدات الرفع مكشوفة للويب عبر server.php
الدليل: server.php السطر 18–35:


<?phpif ($uri !== '/' && file_exists($file) && !is_dir($file)) {    // يُقدِّم الملف مباشرة إذا كان موجوداً    readfile($file);    exit;}
التفسير: أي ملف موجود في المسار يُقدَّم مباشرة. مجلد storage/attachments/ يحتوي على مرفقات المستخدمين. إذا رفع أحدهم ملف .php:

upload-attachment.php يحفظ الملف في storage/attachments/ بالامتداد الأصلي.
server.php يُقدِّمه كملف ثابت (ليس PHP execution في هذه الحالة مع readfile).
لكن الخطر الأكبر: ملف storage/attachments/ يحتوي أسماء ملفات حقيقية كـ"خطاب ضمان بنك الراجحي.pdf" — هذه مستندات قانونية سرية يمكن تخمين مساراتها أو listing مجلدها.

الدليل على غياب الحماية:


<?php// upload-attachment.php السطر 74-81$uploadDir = __DIR__ . '/../storage/attachments/';// لا تحقق من نوع الملف// لا قائمة بيضاء للامتدادات المسموح بها$ext = pathinfo($file['name'], PATHINFO_EXTENSION);$safeName = uniqid() . '_' . preg_replace('/[^a-zA-Z0-9_-]/', '', pathinfo($file['name'], PATHINFO_FILENAME)) . '.' . $ext;
لا قائمة بيضاء للامتدادات.
لا تحقق من نوع MIME الحقيقي (يعتمد على $file['type'] الذي يُرسله العميل — قابل للتزوير).
لا حد أقصى لحجم الملف في الكود.
الخطورة: 🔴 عالية — ملفات خطيرة قد تُرفع وتُتيح تنفيذ أوامر.
التخفيف: إضافة قائمة بيضاء للامتدادات + فحص MIME حقيقي عبر finfo_file() + حظر تقديم محتوى storage/ كملفات قابلة للتنفيذ.

إيجاد-متصفح-02: XSS في record-form.php
