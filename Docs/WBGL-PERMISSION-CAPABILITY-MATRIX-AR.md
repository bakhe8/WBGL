# مصفوفة التحكم الشامل بالصلاحيات والسلوكيات (WBGL)

هذا الملف يعرّف **ما يمكن التحكم فيه فعليًا** لكل صلاحية، على مستويين:

1. `الرؤية`: ما يظهر/يختفي للمستخدم في الواجهة.
2. `التعامل`: ما يُسمح بتنفيذه فعليًا عبر APIs وسلوك النظام.

## ملاحظات تنفيذية

- نظام الصلاحيات يعمل بصيغة:
  - دور المستخدم (Role permissions)
  - ثم overrides لكل مستخدم (`تلقائي/سماح/منع`) من شاشة إدارة المستخدمين.
- واجهات HTML الحرجة (Timeline/History) محمية أيضًا على مستوى API، وليس فقط إخفاء UI.
- صفحة إدارة المستخدمين تعرض الآن كل صلاحية مع:
  - المجال
  - نوع التحكم (`رؤية`/`تنفيذ`/`سلوك واجهة`)
  - سطح التأثير
  - وصف السلوك
- تمت إضافة إدارة أدوار كاملة داخل `views/users.php`:
  - مستوى `Role`: صلاحيات كاملة للدور عبر `api/roles/create.php`, `api/roles/update.php`, `api/roles/delete.php`.
  - مستوى `User`: overrides لكل مستخدم (`allow/deny/auto`) فوق صلاحيات الدور.
